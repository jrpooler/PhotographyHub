# Updated on 03/22/2026 @ 10:00 AM

# Prepare media & target folder
- Use your MacBook to import photos to Bridge 
- Insure SanDisk is attached to MacBook
- Remove SD card from camera after shooting photos
- Insert SD card to Mac - wait for it to show icon on the desktop
- The SD Icon should be named (like NIKON Z 7_2)

# Invoke Bridge to get photos from SD card
- If the 'Photography Workflow Hub' is not open then double click on 'Bridge Tools Website'
- Double click on  'Open Bridge (Shot Dates: Pick Year)' in the HUB tools
- Click your year choice, then click 'OK' button
- Click 'File' at top row of your screen
- Click 'Get Photos From Camera' - this will bring up Adobe Bridge - Photo Downloader
- Set 'Get Photos from: (like NIKON Z7_2) - the name after the camera sd icon   
- Insure 'Create Subfolders' is set to 'Shot Date (yyyymmdd)'
- Insure 'Rename Files:' is set to 'Do not rename files'
- Insure only 'Convert to DNG" is checked
- Insure Template to Use is set to 'Basic Metadata'
- Photos should be visible and checked
- Single Click 'Get Media' at the bottom right of the form (wait for it to finish)
- Eject Camera SD Card from the desktop
- Pull out the SD card from MacBook

# Now use Bridge to tag photos for the new photos just imported
- Open the Shotdate folder: double click that folder
- Select all photos in the shotdate folder
- Go to 'Keywords' in task bar (second row from the top)
- Add a new keyword or use an existing one (to add right click in Keywords panel)
- Repeat this process for each Shotdate folder that was in the camera's sd card

# Now send the tagged photos to the SandDisk
- Open Bridge Tools Website (if not already open)
- Double click 'Run' in 'Send Photos to SanDisk'  (now your imports are in 2 places)
- Check and then Close result panel 
- It is now safe to format the camera's sd card using your camera
- Now your imports are in 3 places: Your MacBook, Your SanDisk, and your camera's backup SD card

# Switch to SanDisk
- Open Bridge Tools Website if not still open
- Double click 'Run' in Open Bridge (Master)
- Go to 'Essentials' (if not already there) 

# Now use Bridge to create a unique Collection
- Click the New Smart Collection icon at the bottom of the Collection list
- Fill in the NewSmartCollection panel: Specify Name of collection
- In source, set 'Look in' to 'Bridge_Photos'
- In Criteria, Keywords contains Keyword name (same as assigned keywords) - one or more keywords, hit +
- If more keywords, add more keywords to criteria
- Add in Criteria, for additional keywords, if any
- Add in Criteria, Rating is greater than or equal to *** (or whatever rating values you specified)
- Results: Match, if all criteria are met
- Checkmark Include All Subfolders
- Checkmark Include Non-indexed Files
- Click 'Find' and wait for search to complete

# Now use your new Bridge collection to rate or delete photos
- Use Bridge Tools Dashboard to Open Bridge (Master)
- Single click your new Collection (in Essentials - Collections panel)
- Single click the first photo in the collection
- If you want to delete this photo, then click Trash Can (in second from top row, right side)
- Otherwise, click a rating (1 to 5) 
- Repeat for each photo in the collection

# Now modify Collection criteria to provide which photos are to be enhanced or edited
- Select the Bridge collection - single click
- Click 'Edit Smart Collection'
- Double Click on '+' in criteria - this will offer a new critera row
- On that row, click 'Rating' , then select 'is greater than or equal to', then select minimum stars (like ***)
- Click 'Find' button
- You can change the number of stars to include or exclude which photos are in the Collection

# Edit your Photos in your Collection using ACR and / or Photoshop

# Setup SmugMug for MacOS - do this section once
- Download and install latest App for MacOS
- Keep new Icon in the dock

# Create a Folder on your Mac to store SmugMug Jpegs - do this once
- For example 'SmugMug-Jpegs' in your Documents folder

# Delete all jpegs currently in your SmugMug-Jpegs folder 

# Setup Bridge
- Using Bridge Tools Website, double click Open Bridge (Master)
- Click the Collection you want to send to SmugMug
- Insure that the filters list only the photos you want to upload
- Select all the photos in the collection
- Right click on one of the selected thumbnails and place cursor on 'Export to'
- Then click 'Custom Export' - this will bring up an export panel

# Fill the panel
- Preset Name: 'Custom Export'
- Save To: 'Specific Folder'
- Click 'Browse' to point to 'SmugMug-Jpegs' in your Documents folder
- Don't use subfolders
- Manage Conflicts: 'Create Unique File Name'
- Image Format: Format JPEG
- Extension: 'jpg'
- Image Quality: Maximum (12)
- Image Resize: Scale Image 100%
- Metadata: Include Original Metadata: All
- Leave Content Credentials alone
- Click 'Start Export' button - wait for completion
- You should now see the jpegs in your Documents / SmugMug-Jpegs folder
- Clear the jobs


# Build SmugMug Gallery
- Click the SmugMug app on the dock
- Click the '+' in front of 'MY SITE'
- Click 'Create Gallery'
- Enter Gallery Name (like Fort Bragg 2026)

- Click 'Upload' on top row
- Click 'INTO NEW GALLERY'
- Select your 'Gallery Preset'
- Click 'Create' button at bottom of template

# Now select all the jpg files in SmugMug-JPEGS folder
- Drag the jpeg photos to the new gallery 
- Wait for all the jpegs to upload
- Click the 3 dots at the bottom of the upload panel 
- Click 'View on SmugMug'

# Remove jpeg photos from SmugMug-JPGEGS (optional)
- Select all jpegs
- Right click: 'Move to Trash'

# Updated on 03/20/2026 @ 5:00 PM

# Prepare GPSMAP 67 -
- Go outside where GPS can lock satellites.
- Wait until GPS gets satellites
- Confirm recording interval (e.g., 1s) and accuracy mode.
- Make sure time zone and date are accurate on camera
- Make sure Daylight Saving Time On or Off is correct
- Run iPhone app for camera that syncs camera date/time to internet time
- Verify time and time zone in camera matches GPS - compare clocks

# Record the activity
- Start activity on GPSMAP 67.
- Walk/drive your route; check track is recording.
- When done taking pictures, end and SAVE the activity; give it a clear name (like: Lake-Tahoe-2025-11-3)

# Initial setup sync with iPhone (Explore App) - DONE ONLY ONCE!
- On GPSMAP, Setup >  Bluetooth > Phone > Connect
- On iPhone, Open Garmin Explore app (paired)
- Activity syncs automatically to Explore > Library > Activities 

# Share as GPX to MacBook AirDrop
- Use iPhone, Explore App should be paired with Garmin Explore
- On iPhone, invoke 'Explore' app
- Push the red + button on bottom of display
- Push 'Saved' at bottom of screen
- Your route should show near the top of the screen - tap it.
- Click red button (Share as GPX) --tap it.
- Export as AirDrop to your nearby MacBook - click the AirDrop icon to your Mac
- Find it in Downloads in Mac - use Finder
- Rename the .gpx file if necessary for easy id (name and date)

# Move gps file from Downloads to Extreme SSD
- Open Extreme SSD / Backup_Files_To_NAS / GPX TrackLogs 
- Move GPX file from Downloads to GPX TrackLogs using 2 finder windows

# Use GPX file to put geodetic data into appropriate Bridge Collection
- Open Bridge
- Select Bridge Collection pertaining to GPX Tracklog
- Select All photos in the collection
- Click Tools > Geotagging
- Click Select File
- Navigate to Extreme SSD > Backup_Files_To_NAS > GPS TrackLogs > Your Tracklog Name
- Click the Start button in GeoTagging panel
- When finished, click Done on Geotagging panel
- Check the Metadata of photos for GPS values

@@ title: Using Legacy Photos (2001–2024) in Collections

@@ summary: Safely use older SanDisk photos in Bridge Collections without moving or breaking anything.

# Updated on 03/05/2026 @ 10:25 AM


# What “Legacy Years” means
- Legacy Years are older photos already stored on the SanDisk (for example 2001–2024).
- These photos live under your main photo root on the SanDisk, usually:
- /Volumes/Extreme SSD/Backup_Files_To_NAS/Bridge_Photos
- You do **not** need to move these into TripPhotoBackup to use them in Collections.
- Collections are like **playlists**: they point to where files live; they do not copy or move the files.

# Before you start
- Make sure the SanDisk (Extreme SSD) is attached to your MacBook.
- Confirm that Bridge Tools is working:
- On the Desktop, double-click **Bridge Tools** to open the Photography Workflow Hub.
- In the hub, make sure the **Tools** row shows buttons such as:
- Open Bridge (Master)
- Build & Open Workflow Hub
- Send Photos to SanDisk
- Open Bridge (Shot Dates: Pick Year)
- Edit MASTER-steps.txt

# Step 1 – Open Bridge at the Legacy photo root
- Attach **Extreme SSD**.
- On the Desktop, double-click **Bridge Tools**.
- In the Photography Workflow Hub, in the Tools row, click:
- **Open Bridge (Master)**
- Bridge 2026 should open pointed at:
- Computer → Extreme SSD → Backup_Files_To_NAS → Bridge_Photos
- If Bridge opens somewhere else:
- In Bridge, click the **Folders** panel.
- Navigate manually to:
- Computer → Extreme SSD → Backup_Files_To_NAS → Bridge_Photos

# Step 2 – Navigate to the year and folder you want
- In the **Folders** panel, expand your Legacy Years inside **Bridge_Photos**.
- Examples:
- Bridge_Photos → 2007 → Mammoth Lakes Trip
- Bridge_Photos → 2013 → Palm Desert
- Bridge_Photos → 2020 → Park City
- Wait for thumbnails to build, especially if this is a large folder.
- Use the **Content** panel to view all photos in the selected folder.

# Step 3 – Select the photos for a Collection
- In the **Content** panel:
- Click a thumbnail to select a single photo.
- Shift-click to select a range of photos.
- Command-click to add or remove individual photos from the selection.
- Take a moment to confirm the selection:
- Are all the right photos highlighted?
- Are any photos selected that you do **not** want in the Collection?

# Step 4 – Create a new Collection from Legacy photos
- Open the **Collections** panel in Bridge:
- If needed, click **Window → Collections**.
- To make a brand-new Collection from your Legacy photos:
- In the Collections panel, click the **+** icon (or use the panel menu → **New Collection**).
- In the dialog that appears:
- Give the Collection a clear name, such as:
- Mammoth Lakes 2007 (Legacy)
- Palm Desert 2013 (Legacy)
- Make sure **Include selected items** is checked.
- Click **OK**.
- Bridge will create the Collection that points to the photos in their existing Legacy folder on the SanDisk.

# Step 5 – Add Legacy photos to an existing Collection
- If you already have a Collection and want to add more Legacy photos:
- Go back to the **Folders** panel and select the legacy folder (for example 2010 → Desert Springs).
- In the **Content** panel, select the additional photos you want.
- In the **Collections** panel:
- Drag the selected photos onto the target Collection name.
- The Collection will now include both the new Legacy photos and anything that was already there.
- Reminder:
- The files are **not** duplicated.
- The Collection simply points to their existing location under Bridge_Photos.

# Step 6 – Verify Collections still work as expected
- In the **Collections** panel, click on your Collection:
- Confirm that thumbnails appear for all expected photos.
- Double-click a photo:
- Confirm it opens in Bridge / ACR / Photoshop as you normally expect.
- If any thumbnails show as missing:
- Check that the photo still lives under:
- /Volumes/Extreme SSD/Backup_Files_To_NAS/Bridge_Photos
- If files were moved outside Bridge, you may need to repair or re-create the Collection.

# Important: Avoid moving Legacy folders unless you plan to rebuild Collections
- Existing Collections remember the **old path** to each file.
- If you move legacy folders (for example from:
- Bridge_Photos/OldArchive/Family
- to
- Bridge_Photos/2005/Family_Trip),
- Bridge will consider those links “broken” and the Collection may show missing files.
- Unless the current folder layout is truly unmanageable:
- Prefer to **leave Legacy Years where they already are** on the SanDisk.
- Use Collections to group and find photos instead of physically moving old folders.

# Optional: If you really must reorganize a Legacy folder
- Before making any big changes, consider:
- Take notes on which Collections use that legacy folder.
- Optionally export a small test folder and experiment with a copy first.
- Basic approach if you decide to reorganize:
- In Finder, **do not** move the real folders yet.
- Instead, test with a smaller subset of photos:
- Copy a few files into a new structure under Bridge_Photos.
- In Bridge, build a new Collection that points to the **new** location.
- Confirm this new structure works for you.
- Only when you are comfortable:
- Plan a larger reorganization and expect to:
- Rebuild or repair existing Collections that referenced the old locations.
- For most cases, it is much safer to:
- Keep Legacy Years where they are.
- Use Collections as your main way to assemble favorite sets of photos.

# Updated on 03/05/2026 @ 10:25 AM


# One-time setup (Dick’s Mac)
- Apple menu > System Settings
- Privacy & Security > Screen & System Audio Recording > FaceTime ON
- Privacy & Security > Accessibility > FaceTime ON

# Start a session (either person can initiate)
- John or Dick starts a FaceTime Audio call
- Dick answers on his Mac (Mac-to-Mac)
- Once connected, Dick shares his screen

# Share screen (Dick)
- Click Share Screen
- Select Entire Screen

# Coaching mode (Dick drives, John watches)
- Dick uses his Mac normally
- John watches and talks Dick through the steps
- John does not request control

# Admin mode (John installs or configures)
- John clicks Request Control
- Dick clicks Allow Control
- If macOS asks for a password, Dick types it on his Mac

# Switch or end session
- Dick can stop control at any time
- Either person can click Stop Sharing or end the FaceTime call

# FaceTime device clarification
- FaceTime calls are sent to Dick’s Apple ID, not a specific device
- If Dick answers on his Mac, the call is Mac-to-Mac
- The iPhone is not involved unless Dick answers on it
